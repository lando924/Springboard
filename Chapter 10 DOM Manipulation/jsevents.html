Events Intro

What is an event? DOM events are "actions" that occur as a result of something the user or the browser does. We can use JS to execute code when these kinds of 'events' happen.

  • clicking on something
  • hovering over something with the mouse
  • pressing certain keys
  • when the DOM has loaded
  • when a form is submitted

MDN has event reference of what can be done. Write code that listens for events. Commonly known as Event Driven Programming.

Event Listeners

    3 options
  • attach the name of the function to the element in HTML
  • attach the name of the function to the element in JS
  • use addEventListener

We don't use the first 2. Rarely use them in general. onclick = 'function'

  • h1.addEventListener("click",function(){ console.log("You just clicked the h1 element!") })
  • easy to remove addEventListener
  • we can add as many addeventlisteners as we want on same object

Load Events

  • waiting for the DOM to load
  • can put script at bottom of body
  • even called DOMContentLoaded
  • easier to put script at bottom
  • it's waiting for the document to be loaded and parsed

Event Object

  • you can pass special object in parameter while running event
  • target-what element is the target of the event
  • pageX/pageY where on the page did this event occur?
  • key what key was pressed that triggered event?
  • preventDefault() a function used to prevent the default behavior of the event

Form Submissions

  • action"" brings you to new page. you can leave empty
  • event.preventDefault() keeps things on single form
  • usually do it for forms

  • trigger if a user would show up as an input
  • 'keydown'
  • 'keypress'

Multiple Events

  • we can loop over them

Event Delegation

  • adding the event listener when we create elements
  • event delegation
  • if we're adding 1,000,000 friends, we have the same amount of listeners
  • we can use event delegation instead
  • in example the parent element is the ul element
  • we can attach a single event listener on the parent or delegate element. if the event happens inside of certain child element we can access that child element using event.target
  • listen for the parent element
  • can have different outcomes for different things using this

Data Attributes

  • store meta data without showing it
  • custom attributes for elements
  • data-
  • read them easily in CSS and Javascript
  • dataset to see all attributes

Recap

  • We can add event listeners using addEventListener and remove them using removeEventListener
  • Using the event object, we can gather useful information about the target, tagName and much more
  • To add element metadata, we can use data attributes and read them using getAttribute or dataset
