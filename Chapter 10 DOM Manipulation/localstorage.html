Goals

  • Utilize localStorage to save information in the browser
  • Compare and contrast localStorage and sessionStorage
  • Add and remove primitives to/from localStorage
  • Add and remove objects to/from localStorage

localStorage

What is an localStorage? localStorage is a mechanism for storing information in the browser for a specific domain. The API is quite easy to use and very minimal.

    localStorage vs sessionStorage
  • data stored in localStorage has no expiration time
  • data stored in sessionStorage gets cleared when the browsing session ends
  • treat it as a database.

modifying localStorage

    Get Set Item
  • setItem method. string form of object. key and a value
  • getItem, is how we retrieve information. pass in a key and retrieve whatever matches with the key
  • parseInt if you want to make value into a number
  • removeItem to remove key/value
  • localStorage.clear clears everything

JSON adding Objects to localStorage

  • if working with an object or array you have to parse into string when in localStorage
  • In order to get back our original data type we need to convert this arraty to a special string format called JSON
  • JSON (JavaScript Object Notation) is a lightweight data-interchange format. It is easy for humans to read and write. easy for machines to parse and generate
  • JSON.stringify-converts JavaScript to JSON or stringify
  • JSON.parse parses a string as JSON
  • convert it to store it, parse it to retrieve it

Recap

  • localStorage is useful for storing info in browser
  • store objects, use JSON.stringify when setting JSON.parse to retrieve
  • if you want to store info for the time a tab is open, use sessionStorage
